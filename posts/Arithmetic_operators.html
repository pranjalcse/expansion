Arithmetic Operators

Arithmetic operators are like the building blocks of math in Java. They are the basic operators used to perform arithmetic calculations like addition, subtraction, multiplication, division, and modulo operation (remainder). All arithmetic operators are binary operators as they operate on two operands. They are denoted by the familiar symbols: +, -, *, /, and %. Let's dive into the arithmetic operators in Java with some examples.

Addition (+)

The addition operator is used to add two or more values together. For example:


public class Example {

    public static void main(String[] args)
    {
        int num1 = 5;
        int num2 = 10;
        int sum = num1 + num2;
        System.out.println("The sum of " + num1 + " and " + num2 + " is " + sum);
    }
}

Output: The sum of 5 and 10 is 15

Subtraction (-)

The subtraction operator is used to subtract one value from another. For example:


public class Example {

    public static void main(String[] args)
    {
        int num1 = 15;
        int num2 = 10;
        int difference = num1 - num2;
        System.out.println("The difference between " + num1 + " and " + num2 + " is " + difference);
    }
}

Output: The difference between 15 and 10 is 5

Multiplication (*)

The multiplication operator is used to multiply two or more values together. For example:


public class Example {

    public static void main(String[] args)
    {
        int num1 = 5;
        int num2 = 10;
        int product = num1 * num2;
        System.out.println("The product of " + num1 + " and " + num2 + " is " + product);
    }
}

Output: The product of 5 and 10 is 50

Division (/)

The division operator is used to divide one value by another. For example:


public class Example {

    public static void main(String[] args)
    {
        int num1 = 10;
        int num2 = 5;
        int quotient = num1 / num2;
        System.out.println("The quotient of " + num1 + " divided by " + num2 + " is " + quotient);
    }
}

Output: The quotient of 10 divided by 5 is 2

Modulus (%)

The modulus operator is used to find the remainder of a division operation. For example:


public class Example {

    public static void main(String[] args)
    {
        int num1 = 10;
        int num2 = 3;
        int remainder = num1 % num2;
        System.out.println("The remainder of " + num1 + " divided by " + num2 + " is " + remainder);
    }
}

Output: The remainder of 10 divided by 3 is 1

When we use the modulo operator (%) with floating point numbers, Java will produce a compile-time error, as the modulo operator can only be used with integer types.
For example, the following code will produce a compile-time error:


public class Example {

    public static void main(String[] args)
    {
        double x = 10.5;
        double y = 3.2;
        double result = x % y; // Compile-time error: bad operand types for binary operator '%'

    }
}

Output: Compile-time error: bad operand types for binary operator '%'

Data Type of Result of Arithmetic Operators in Java

In Java, the data type of the result of an arithmetic operation depends on the data types of the operands involved in the operation. When arithmetic operators are used with operands of different data types, Java automatically promotes the operands to a higher data type before performing the operation. This is known as type promotion. In Java, type promotion follows a set of rules that determine the resulting data type after the promotion. These rules are as follows:

  1. If either operand is a double, the other operand is promoted to a double, and the result is a double.
  2. If either operand is a float and the other operand is not a double, both operands are promoted to float, and the result is a float.
  3. If either operand is a long and the other operand is not a double or a float, both operands are promoted to long, and the result is a long.
  4. Otherwise, both operands are promoted to int, and the result is an int.

Let's look at some examples to better understand type promotion with arithmetic operators:


public class Example {

    public static void main(String[] args)
    {
        int num1 = 10;
        float num2 = 3.5f;
        float result = num1 + num2; // num1 is promoted to float, and the result is a float

        long num1 = 1000000000L;
        int num2 = 5;
        long result = num1 * num2; // num2 is promoted to long, and the result is a long

        short a = 10;
        byte b = 2;
        int result = a * b; // result will be an int with value 20

        int a = 7;
        int b = 2;
        int result = a / b; // result is 3, not 3.5

        int a = 7;
        int b = 2;
        double result = a / b; // result will be a double with value 3.0
    }
}

In this example, we have two integer variables a and b with values 7 and 2 respectively. We are then performing an arithmetic operation of division using the / operator. Since both operands are integers, Java performs integer division and the result will be the quotient of a divided by b, which is 3.

However, we are storing the result in a double variable named result. Java performs an implicit type conversion, also known as type promotion, to convert the integer result to a floating-point value. In this case, the integer 3 is converted to a double value 3.0. If we want to obtain a fractional result, we need to make sure that at least one of the operands is a floating-point type. For example:


public class Example {

    public static void main(String[] args)
    {
        double x = 7.0;
        int y = 2;
        double result = x / y; // result is 3.5
    }
}

In this example, we have one operand of type double and one operand of type int. When we divide them, the result is a double, and the fractional part is preserved. Another way to obtain a fractional result is to cast one of the operands to a floating-point type before performing the division:


public class Example {

    public static void main(String[] args)
    {
        int a = 7;
        int b = 2;
        double result = (double) a / b; // result is 3.5
    }
}

In this example, we cast the integer a to a double before dividing it by b. This forces the result to be a double, and the fractional part is preserved.
